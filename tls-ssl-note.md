# TLS/SSLのメモ

なんか自信がなくなってきたんで、もういちど整理

まずこれ

1. CAの公開鍵
2. サーバの証明書(サーバの公開鍵をCAの秘密鍵で署名したもの)
3. サーバの秘密鍵

ハンドシェイク

### 1. ハンドシェイク開始

- Client → Server: `ClientHello` (TLSのバージョン・暗号スイート一覧・乱数)

### 2. サーバ証明書送付

- Server → Client: `ServerHello` サーバ証明書(=サーバ公開鍵+CA署名)

### 3. 証明書検証(クライアント側)

- クライアント:
  - **CA公開鍵**で証明書署名を検証  
    → 改ざんなし・正当サーバと確認
  - 証明書内の**サーバ公開鍵取得**

### 4. 共通鍵(セッション鍵)確立

※ 代表例(RSA型):

- Client:
  - プレマスターシークレット生成
  - **サーバ公開鍵で暗号化**
  - → Server に送信

- Server:
  - **サーバ秘密鍵で復号**
  - 同じプレマスター取得

→ 双方で同じセッション鍵生成

### 5. 暗号通信開始

- 以降:
  - 対称鍵(セッション鍵)で暗号化通信

## ECDHE / DHE(今主流)

RSAは「公開鍵で暗号」ができるけどいまは主流じゃない。

RSA以外では **鍵交換アルゴリズムで共通鍵作る方式**

TLS 1.3 では RSAは使えない

### 流れ

1. ClientHello
2. ServerHello + **サーバ公開パラメータ(ECDHEの公開値)** + 証明書
3. クライアント:
   - CA公開鍵で証明書検証
   - サーバ公開パラメータ取得
4. Client → Server:
   - **自分の公開パラメータ**送信
5. 双方:
   - 自分の秘密パラメータ × 相手公開パラメータ  
     → **同じ共有秘密生成**

## サーバ公開パラメータ

**ECDHE文脈の「サーバ公開パラメータ」= 鍵交換用の公開情報セット**

**サーバ公開パラメータ = 曲線情報 + サーバ一時公開鍵 + その署名**

### 中身(ECDHE)

- **楕円曲線名**  
  例: `secp256r1`, `x25519`

- **ベースポイント(G)**  
  曲線上の基準点(実装により固定 or 暗黙)

- **サーバの公開値(公開鍵)**

  ```
  Qs = ds × G
  ```

  - `ds`: サーバ秘密値(セッションごと生成)
  - `Qs`: これをクライアントに送る

- **署名**
  - 上記一式を**サーバ証明書の秘密鍵で署名**
  - 改ざん・MITM防止

### 何のためか

- クライアントが自分の公開値 `Qc` 作る
- 双方で計算:
  ```
  共有秘密 = ds × Qc = dc × Qs
  ```
- 同じ値になる

### RSAとの対比

- RSA: 公開鍵 = 暗号化用
- ECDHE: 公開値 = **計算用(鍵生成の材料)**

### 重要ポイント

- 公開パラメータだけでは秘密鍵出ない
- 毎回 `ds` 変える → Forward Secrecy
- 証明書の鍵は「署名専用」

- TPMは遅い → ハンドシェイク重い
- RSA key exchange自体は旧式(TLS1.3で消滅)

## 構成

- CAの公開鍵
- サーバ公開鍵 → 証明書に入ってる(通常どおり)
- サーバ秘密鍵 → **TPM内(外に出ない)**

### 1. ClientHello

通常

### 2. ServerHello + 証明書

- サーバ → クライアント
  - 証明書(公開鍵)

### 3. クライアント検証

- CA公開鍵で証明書検証
- サーバ公開鍵取得

### 4. プレマスター送信

- クライアント:
  - プレマスター生成
  - **サーバ公開鍵で暗号化**
  - → Server

### 5. サーバ復号 (ここがTPM.変わるのはここだけ)

- Server:
  - 受信した暗号文を**TPMに渡す**
  - TPM:
    - 内部の**サーバ秘密鍵で復号**
    - プレマスター取り出し
  - ※秘密鍵は外に出ない

### 6. セッション鍵生成

- 双方:
  - プレマスター + ランダム値 → セッション鍵

### 7. 暗号通信開始

対称鍵通信

### 利点

- 秘密鍵抽出不可
- 侵入されても鍵漏れにくい
- HSM(Hardware Security Module)の軽量版
